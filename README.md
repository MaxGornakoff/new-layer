# Filament Shop

Интернет-магазин филамента для 3D-печати: Vue 3 + Laravel API + MySQL.

## Стек

- **Frontend:** Vue 3, Vue Router, Pinia, Vite, Tailwind CSS v4
- **Backend:** Laravel 13, Sanctum (API tokens)
- **БД:** MySQL 8.4
- **Локально:** Docker (nginx + PHP 8.4 + MySQL)

## Структура

```
backend/     Laravel API (document root: backend/public)
frontend/    Vue SPA
docker/      nginx и PHP Dockerfile
```

Один домен в продакшене: Vue собирается в `backend/public`, API — под `/api`.

## Быстрый старт

### 1. Backend (.env)

```bash
cp backend/.env.docker backend/.env
```

Сгенерируйте ключ (через Docker после шага 2):

```bash
docker compose run --rm php php artisan key:generate
```

### 2. Docker

```bash
docker compose up -d --build
docker compose exec php php artisan migrate --seed
```

API: http://localhost:8080/api/health

### 3. Frontend (разработка)

```bash
cd frontend
npm install
npm run dev
```

Сайт: http://localhost:5173 (прокси `/api` → `:8080`)

### 4. Production-сборка фронта

```bash
cd frontend
npm run build
```

Статика попадёт в `backend/public` (рядом с `index.php` Laravel).

## Деплой на хостинг

Полная инструкция (IHC / ISPmanager, медиа, UTF-8, цикл обновлений): **[DEPLOY.md](./DEPLOY.md)**.

Кратко:

1. `cd frontend && npm run build` → статика в `backend/public`
2. Залить код на сервер **без** перезаписи прод-`.env` и `storage/app/public`
3. `/opt/alt/php83/usr/bin/php artisan migrate --force` и `config:clear`
4. На IHC (document root = корень Laravel): в `.env` — `FILESYSTEM_PUBLIC_PATH=/storage/app/public`

### Параллельно с наполнением сайта клиентом

- **Да:** клиент заполняет товары/категории/картинки на проде; вы пишете код локально и потом деплоите PHP/Vue + **миграции**. Данные клиента не трогаются, если не заливать дамп БД и не затирать `storage/app/public`.
- **Да:** обычный цикл — локальная разработка → `npm run build` → залить файлы → `migrate --force`. Как раньше, только без повторного импорта локальной БД на прод.
- **Нет:** повторный `mysqldump` с локалки на прод — это затрёт контент клиента.

Прод = источник правды по контенту. Локально = код и схема (миграции).

## Текущий фокус и отложенные задачи

> **Первым делом проверить:** **Яндекс Доставку (ЯД)** и **Желдорэкспедицию (ЖДЭ)** — код интеграций готов, но на реальных ключах клиента ещё не прогоняли end-to-end. Байкал и Деловые линии уже проверены и работают.

**Чеклист проверки ЯД и ЖДЭ:**
1. Админка → **Доставка** (`/admin/delivery`) — включить службу, сохранить ключи, заполнить пункт отправления
2. Нажать **«Проверить подключение»** для каждой службы
3. Checkout — город → тариф → ПВЗ → оформление заказа
4. **ЯД:** нужен OAuth-токен и `platform_station_id` (ID склада из ЛК Яндекс Доставки) — у клиента ID склада пока нет, без него полный расчёт не проверить
5. **ЖДЭ:** логин и API-токен из [i.jde.ru](https://i.jde.ru) (в поле «Пароль» — токен, не пароль от сайта); желательно указать **ID терминала отправления**

**Сейчас в приоритете (после проверки ЯД и ЖДЭ):**
- Логотипы перевозчиков в checkout вместо текстовых радиокнопок
- СДЭК — когда появятся ключи
- Страница «Заказ оформлен» с номером заказа
- Полировка UX корзины/checkout (сверка цен/остатков с сервером)

**Отложено на потом (углублённые доработки):**
- Оформление заказа: сверка цен/остатков с сервером на checkout
- Сохранение в заказе структурированных данных доставки (служба, ПВЗ, тариф), а не только строки адреса
- Онлайн-оплата
- Прочие улучшения воронки покупки (гость vs авторизация, полировка UX)
- Push / звук при новом заказе (сейчас — бейдж и баннер в админке)

**Уже работает:**
- **Доставка — фундамент:** сервисный слой `backend/app/Services/Delivery/`, таблица `delivery_settings`, админка `/admin/delivery`
- **Админка — грузоместо:** параметры упаковки на N катушек (ширина, длина, высота, вес в см/кг), склад по умолчанию, пункт отправления отдельно для каждой ТК; ключи API в БД в зашифрованном виде; сворачиваемые блоки служб
- **Байкал Сервис:** расчёт, поиск городов (FIAS), ПВЗ, кнопка «Проверить подключение»; тестовый API через `BAIKAL_USE_TEST_API=true` — **проверено**
- **Деловые линии:** расчёт (в т.ч. с terminal ID), КЛАДР-города, терминалы/ПВЗ, срок из `orderDates`, проверка подключения — **проверено**
- **Почта России:** тарифный API, индекс после выбора службы, debounce ввода индекса, резолвер город→индекс — код готов
- **Яндекс Доставка / Желдорэкспедиция:** код готов, нужна проверка на ключах клиента
- **Checkout:** город → тарифы → служба → ПВЗ → итог; в заказе раздельно ТК и адрес ПВЗ; вёрстка 2 колонки
- **Уведомления:** `AppToast` (глобальные + рядом с полем при валидации форм); баннер cookie-согласия; согласие на ПДн при регистрации; юридические stub-страницы
- **Цвет товара:** hex-палитра в админке и фильтре каталога (свайчи)
- **Главная:** hero (адаптив), категории, статистика, ленты товаров, «О нас» + преимущества, футер; якорь `/#about-us` с докруткой после загрузки полок (`useHashScroll`)
- **Каталог:** мультифильтры, мобильная шторка (свайп закрытия), сетка 2 колонки, единые карточки с главной
- **Страница товара:** макет 1440px, галерея с миниатюрами, синий бейдж цены, адаптив
- **Мобильное меню:** шторка справа (как фильтры), свайп закрытия, без сдвига контента
- **Шапка:** sticky с отступом при скролле; мини-корзина и профиль по hover; мобильный поиск на всю ширину
- **Корзина / мини-корзина:** синие `−` / кол-во / `+` как на витрине; CTA «К оформлению» / «Добрать в корзине» по кратности ×10; адаптивные карточки на мобилке
- **ЛК клиента:** редактирование профиля; заказы с общей шапкой колонок и раскрытием состава
- **Админка — UX форм:** форма добавления скрыта → кнопка «Добавить…»; режим правки с баннером; иконки «Изменить» / «Удалить» вместо текста
- **Админка — заказы:** фильтр по статусам + «Все»; удаление; бейдж новых на обзоре и баннер в сайдбаре; цветные статусы; сортировка
- **Товары — медиа и цены:** галерея, видео, compare-at; `ProductPrice`
- Динамическое меню, поиск, прелоадеры, Tailwind-first, `cursor: pointer` на интерактиве
