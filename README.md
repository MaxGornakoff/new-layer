# Filament Shop

Интернет-магазин филамента для 3D-печати: Vue 3 + Laravel API + MySQL.

## Стек

- **Frontend:** Vue 3, Vue Router, Pinia, Vite, Tailwind CSS v4
- **Backend:** Laravel 13, Sanctum (API tokens)
- **БД:** MySQL 8.4
- **Локально:** Docker (nginx + PHP 8.4 + MySQL)

## Структура

```
backend/     Laravel API (document root: backend/public)
frontend/    Vue SPA
docker/      nginx и PHP Dockerfile
```

Один домен в продакшене: Vue собирается в `backend/public`, API — под `/api`.

## Быстрый старт

### 1. Backend (.env)

```bash
cp backend/.env.docker backend/.env
```

Сгенерируйте ключ (через Docker после шага 2):

```bash
docker compose run --rm php php artisan key:generate
```

### 2. Docker

```bash
docker compose up -d --build
docker compose exec php php artisan migrate --seed
```

API: http://localhost:8080/api/health

### 3. Frontend (разработка)

```bash
cd frontend
npm install
npm run dev
```

Сайт: http://localhost:5173 (прокси `/api` → `:8080`)

### 4. Production-сборка фронта

```bash
cd frontend
npm run build
```

Статика попадёт в `backend/public` (рядом с `index.php` Laravel).

## Деплой на хостинг

1. Собрать фронт: `npm run build` в `frontend/`
2. Залить `backend/` на сервер (включая `vendor/`, если нет Composer)
3. Document root → `backend/public`
4. Настроить `.env` (MySQL, `APP_URL`, `APP_DEBUG=false`)
5. Выполнить `php artisan migrate --seed` и `php artisan storage:link`

## Текущий фокус и отложенные задачи

> **Первым делом проверить:** **Яндекс Доставку (ЯД)** и **Желдорэкспедицию (ЖДЭ)** — код интеграций готов, но на реальных ключах клиента ещё не прогоняли end-to-end. Байкал и Деловые линии уже проверены и работают.

**Чеклист проверки ЯД и ЖДЭ:**
1. Админка → **Доставка** (`/admin/delivery`) — включить службу, сохранить ключи, заполнить пункт отправления
2. Нажать **«Проверить подключение»** для каждой службы
3. Checkout — город → тариф → ПВЗ → оформление заказа
4. **ЯД:** нужен OAuth-токен и `platform_station_id` (ID склада из ЛК Яндекс Доставки) — у клиента ID склада пока нет, без него полный расчёт не проверить
5. **ЖДЭ:** логин и API-токен из [i.jde.ru](https://i.jde.ru) (в поле «Пароль» — токен, не пароль от сайта); желательно указать **ID терминала отправления**

**Сейчас в приоритете (после проверки ЯД и ЖДЭ):**
- Логотипы перевозчиков в checkout вместо текстовых радиокнопок
- Полировка главной: секция «О нас» и прочие блоки по макету
- СДЭК — когда появятся ключи

**Отложено на потом (углублённые доработки):**
- Оформление заказа: сверка цен/остатков с сервером на checkout
- Страница «Заказ оформлен» с номером заказа
- Сохранение в заказе структурированных данных доставки (служба, ПВЗ, тариф), а не только строки адреса
- Онлайн-оплата
- Прочие улучшения воронки покупки (гость vs авторизация, полировка UX)

**Уже работает:**
- **Доставка — фундамент:** сервисный слой `backend/app/Services/Delivery/`, таблица `delivery_settings`, админка `/admin/delivery`
- **Админка — грузоместо:** параметры упаковки на N катушек (ширина, длина, высота, вес в см/кг), склад по умолчанию, пункт отправления отдельно для каждой ТК; ключи API в БД в зашифрованном виде; сворачиваемые блоки служб
- **Байкал Сервис:** расчёт, поиск городов (FIAS), ПВЗ, кнопка «Проверить подключение»; тестовый API через `BAIKAL_USE_TEST_API=true` — **проверено**
- **Деловые линии:** расчёт (в т.ч. с terminal ID), КЛАДР-города, терминалы/ПВЗ, срок из `orderDates`, проверка подключения — **проверено**
- **Яндекс Доставка:** `location/detect`, `pickup-points/list`, `pricing-calculator`; ПВЗ и пересчёт на checkout; тест подключения; прод-хост `b2b-authproxy.taxi.yandex.net`, тест — `YANDEX_DELIVERY_USE_TEST_API=true` — **код готов, нужна проверка на ключах клиента** (блокер: `platform_station_id` склада)
- **Желдорэкспедиция:** API `https://api.jde.ru/vD` — города (`geo/SearchCity`), ПВЗ (`geo/search`), калькулятор (`calculator/price`); кэш справочников 24 ч; пересчёт с ПВЗ на checkout; тест подключения — **код готов, нужна проверка на ключах клиента**
- **Checkout:** город → тарифы всех включённых ТК → выбор службы → ПВЗ → итог; адрес доставки собирается автоматически; вёрстка 2 колонки (контакты | доставка + итог)
- **Главная (hero):** слайдер под шапкой из админки (`/admin/hero-slides`), HTML в заголовке/подзаголовке, кнопка со стрелкой, свайп и перетаскивание мышью, пагинация-точки, автопрокрутка с интервалом из админки, зацикливание
- **Главная (цифры):** блок `HeroDigits` поверх слайдера — анимация набегания цифр от 0 при появлении в viewport
- **Главная (категории):** блок `HomeCategories` — карточки по макету, преимущества, градиентный фон из 2 цветов (админка), белый круг 20%, клик → каталог с фильтром
- **Главная (статистика):** блок `HomeStats` — показатели компании под категориями
- **Главная (ленты товаров):** `HomeProductShelves` — карусели/сетки товаров по категориям, заголовок-«пилюля» со ссылкой «Смотреть все», синие кнопки корзины (`CartProductQuantityControl variant="shelf"`) — «В корзину» с иконкой корзины, затем `−` / ввод / `+`
- **Товары — медиа:** несколько изображений (`products.images`), видео (`products.video`); админка — галерея с порядком, добавление/удаление; `ProductImagesSlider` в каталоге, на странице товара и в лентах (1 фото — статично, 2+ — слайдер; hover-зоны на десктопе, свайп на мобиле)
- **Товары — цены:** «старая цена» для витрины — точная сумма или наценка в % (`compare_at_price`, `compare_at_markup_percent`); приоритет у точной цифры; компонент `ProductPrice` с перечёркнутой ценой
- **Админка:** единый UI для всех разделов — заголовки страниц, секции форм, карточки списков, обзор с быстрыми ссылками (`/admin`)
- **Стилизация:** правило Tailwind-first (`.cursor/rules/frontend-tailwind.mdc`) — утилиты в приоритете, scoped CSS только по необходимости
- Динамическое меню из админки, поиск в шапке, мини-корзина и меню профиля по hover (десктоп)
- **Корзина и количество:** `+` / `−` / поле ввода синхронизируются с корзиной напрямую (без отдельной кнопки «В корзину»); debounce 1 с при ручном вводе + мгновенное применение по blur/Enter; правило заказа кратно 10 катушкам (сумма по всей корзине) — на фронте и в API
- Корзина: тосты, `localStorage`, лимит по остатку, страница корзины, мини-корзина, checkout с расчётом доставки
- **Личный кабинет:** «Мои заказы» — раскрываемые карточки с составом заказа, количествами и ссылками на товары
- Прелоадеры (`AppLoader`) на страницах с загрузкой данных с API
